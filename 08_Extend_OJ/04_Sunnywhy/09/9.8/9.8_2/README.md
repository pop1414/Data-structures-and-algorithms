# 树的最小带权路径长度

## 题目描述

对一棵`n`个结点的树（结点编号为从`0`到`n-1`，根结点为`0`号结点，每个结点有各自的权值）来说：

1.  结点的路径长度是指，从根结点到该结点的边数；
2.  结点的带权路径长度是指，结点权值乘以结点的路径长度；
3.  树的带权路径长度是指，树的所有叶子结点的带权路径长度之和。

现有`n`个不同的正整数，需要寻找一棵树，使得树的所有叶子结点的权值恰好为这`n`个数，并且使得这棵树的带权路径长度是所有可能的树中最小的。求最小带权路径长度。

## 输入描述

第一行一个整数`n`（`1 ≤ n ≤ 50`），表示正整数的个数；
第二行为用空格隔开的`n`个正整数（每个正整数均不超过`1e8`），含义如题所示。

## 输出描述

输出一个整数，表示最小带权路径长度。

## 样例

### 样例输入

```
3
1 2 9
```

### 样例输出

```
15
```

### 样例解释

对应最小带权路径长度的树如下所示，其中黑色数字为结点编号，编号右下角的格式为`结点权值*结点路径长度=结点带权路径长度`。由此可得树的带权路径长度为 `2 + 4 + 9 = 15`，是所有可能的树中最小的。

```
    0
   / \
  1   2 (9*1=9)
 / \
3   4 (1*2=2) (2*2=4)
```

## 感悟

- 哈夫曼树是一种逻辑结构（我们需要的是 “每次选最小两个合并” 的树型逻辑）；
- 而物理实现方法有：`指针式的二叉树（left/right）`；`优先队列（PQ）`
- 这体现了逻辑结构不一定与物理实现一样，哈夫曼树是树形结构，而我们使用PQ实现更快
- 编程中 `实现逻辑` 比 `复刻物理结构` 更重要。我们用PQ实现哈夫曼树的逻辑，而不是用指针式的二叉树复刻
