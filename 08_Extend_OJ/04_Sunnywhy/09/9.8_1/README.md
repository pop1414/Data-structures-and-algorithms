# 合并果子

通过数 1401 提交数 2378 难度 中等 显示标签 ☆

## 题目描述

有 `n` 堆果子，每堆果子的质量已知，现在需要把这些果子合并成一堆，但是每次只能把两堆果子合并到一起，同时会消耗与两堆果子质量之和等值的体力。显然，在进行 `n-1` 次合并之后，就只剩下一堆了。为了尽可能节省体力，需要使耗费的总体力最小。求需要耗费的最小总体力。

## 输入描述

第一行一个整数 `n`（`1 ≤ n ≤ 100`），表示果子的堆数；
第二行为用空格隔开的 `n` 个正整数（每个正整数均不超过 `100`），表示每堆果子的质量。

## 输出描述

输出一个整数，表示需要耗费的最小总体力。

## 样例 1

### 输入

```
3
1 2 9
```

### 输出

```
15
```

### 解释

先将质量为 `1` 的果堆和质量为 `2` 的果堆合并，得到质量为 `3` 的果堆，同时消耗体力值 `3`；
接着将质量为 `3` 的果堆和质量为 `9` 的果堆合并，得到质量为 `12` 的果堆，同时消耗体力值 `12`；
因此共消耗体力值 `3 + 12 = 15`。

## 解题思路

- 这是典型的最小合并代价问题，本质是哈夫曼树/WPL
- 每次合并当前最小的两堆，累加代价（WPL），用最小堆实现

## 实现要点

- 用 `priority_queue` + `greater<>` 作为小根堆。
- `n=1`时，WPL = 0。因为一堆不需要再操作它
- 代价可能累加，所以用`long long`更保险

## 复杂度

- 时间: `O(nlogn)`
- 空间: `O(n)`

## 易错点

- 进行累加循环的时候，一定要保证`min_heap`中至少有两堆，只剩下一堆的时候，循环停止
- 所以用`while(min_heap.size() >= 2)`
